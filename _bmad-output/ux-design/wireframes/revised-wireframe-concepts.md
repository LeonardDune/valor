# Revised CAUSA Wireframe Concepts

Based on the comprehensive UX specification, here are the key revisions to the wireframes:

## CAUSA Positioning Change
**OLD:** Traditional causal modeling tool with drag-and-drop diagrams
**NEW:** Collaborative reasoning environment where causal understanding emerges from conversation

## Dashboard Revisions

### Header
- **OLD:** "CAUSA Dashboard" - generic workspace view
- **NEW:** "CAUSA - Collaborative Reasoning Environment" - emphasizes the social, collective nature

### Primary Action
- **OLD:** "+ New Workspace" - technical artifact creation
- **NEW:** "Start New Reasoning Session" - social collaborative activity

### Main Content Areas
- **OLD:** "Recent Workspaces" - individual artifacts
- **NEW:** "Active Reasoning Sessions" - live collaborative processes

### Session Cards
- **OLD:** Modified date, collaborator count - artifact metadata
- **NEW:** Current focus question, collective progress indicators, active participants

### Templates
- **OLD:** "Quick Start Templates" for technical setups
- **NEW:** "Reasoning Patterns" for different collaborative problem types (exploratory, analytical, evaluative)

## Main Workspace Revisions

### Overall Layout
- **OLD:** Modeling tools sidebar + canvas + metadata panel
- **NEW:** Conversation-first with semantic spatialization

### Left Sidebar
- **OLD:** "Modeling Tools" (Add Variable, Add Relationship)
- **NEW:** "Reasoning Tools" (Add Claim, Add Perspective, Question Assumption)

### Canvas
- **OLD:** "Causal Model Canvas" - diagramming surface
- **NEW:** "Shared Reasoning Space" - semantic spatialization where proximity indicates epistemic relationships

### Right Panel
- **OLD:** "AI Agent Suggestions" - external AI input
- **NEW:** "Collective Insights" - AI suggestions integrated into conversation flow

### Bottom Panel
- **OLD:** "Model Metadata & Uncertainty" - technical tracking
- **NEW:** "Collective Understanding Status" - social progress indicators

## Settings Revisions

### Overall Focus
- **OLD:** Workspace technical settings
- **NEW:** Reasoning session social norms and collective standards

### Key Settings Areas
- **OLD:** Privacy (Public/Private), technical integrations
- **NEW:** Collective norms (disagreement handling, contribution expectations), epistemic standards (uncertainty thresholds, validation requirements)

## Design Principle Applications

### Conversational Causality
- Interface elements support dialogue flow over diagram creation
- Claims appear as conversation contributions, not diagram nodes

### Collective Sensemaking
- All UI elements emphasize "we" over "I"
- Progress indicators show group understanding evolution

### Uncertainty as Asset
- Visual design treats disagreement as productive, not problematic
- Provisional states are visually comfortable, not flagged as incomplete

### Automatic Structure
- No visible "modeling" actions - structure emerges invisibly from use
- Users focus on reasoning, system handles representation

### Web-Native Collaboration
- All interactions designed for shared, persistent collaborative spaces
- No individual "save" concepts - everything is collective

## Emotional Design Integration

### Safety First
- Provisional states as default (no red "incomplete" indicators)
- Easy social retreat from claims
- Collective ownership reduces personal risk

### Productive Friction
- Disagreement visualization as connection opportunities
- Parallel claim display encourages exploration

### Calm Cognitive Load
- Progressive disclosure prevents overwhelm
- Familiar conversation patterns
- Invisible complexity management

## Implementation Priorities

1. **Dashboard:** Transform from workspace manager to session facilitator
2. **Main Workspace:** Redesign as conversation-first reasoning environment
3. **Settings:** Focus on collective norms over technical configuration

These revisions position CAUSA as infrastructure for collective causal thinking, not a tool for individual model creation.
