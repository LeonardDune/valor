# Technical Design Document: Dashboard 2.0
## 1. Executive Summary
*   **Feasibility:** High. The Neo4j graph structure naturally supports the proposed "multidimensional" navigation better than the current hierarchy.
*   **Complexity Score:** 7/10. Requires significant frontend responsiveness and backend query refactoring, but fits the core technology stack.
*   **Core Approach:** Shift from `Organization -> Project -> Theme` drill-down to a `User -> Themes` query with metadata aggregation. Implement "Perspectives" as predefined filter configurations on the Graph View, not distinct DB entities.

## 2. Design Intent (Architecture)
### Conceptual Shift
*   **Old:** "Where is this file?" (Folder-like structure).
*   **New:** "What am I working on?" (Context-driven).

### Perspectives Pattern
"Perspectives" (CAUSA, NORM, ACTOR) will be implemented as **View Configurations** within the Theme context.
*   **Backend:** No new "Perspective" entity. Uses existing graph nodes (`Claim`, `Factor`) but the *retrieval queries* will be scoped by these perspectives (e.g., CAUSA = only Causal claims).
*   **Frontend:** The mapped "Perspective icons" will trigger specific `graph_mode` states in the `Workspace` component.

### Data Flow
1.  **Dashboard Load:** `GET /dashboard/activity` (New) -> Fetches flat list of user's active themes with "Badges" (Org/Project name).
2.  **Navigation:** User clicks Theme Card -> Routes to `/theme/:id`.
3.  **Perspective Deep Link:** User clicks "CAUSA" on card -> Routes to `/theme/:id?mode=CAUSA` -> `Workspace` initializes in Causal Analysis mode.

## 3. Impact Analysis (The "Touch List")

### 📂 Database / Schema
*   **No new tables.**
*   **New Queries (`backend/app/db/dashboard.py`):**
    *   `get_active_themes_for_user(user_email)`: Returns themes linked via `MEMBER_OF` path, ordered by activity/recency.
    *   `get_theme_perspectives_status(theme_id)`: (Optional) Counts stats for each perspective (e.g., "5 Causal Claims").

### ⚙️ Backend Services
*   **`backend/app/routers/dashboard.py`:** Add `GET /themes` endpoint.
*   **`backend/app/routers/workspace.py` (New?):** To handle perspective-specific data booting.

### 🌐 API / Interfaces
*   `GET /dashboard/themes?filter=active`
*   `GET /dashboard/perspectives` (Static config or dynamic stats?)

### 🎨 Frontend / UI
*   **`DashboardLayout.tsx`:** Complete overhaul. Remove Sidebar hierarchy, add "Context Panel" (Right side).
*   **`ThemeCard.tsx` (New):** Rich card component.
*   **`Sidebar.tsx`:** Refactor to "Global Navigation" (Icons only).
*   **`App.tsx`:** Update routes. `/dashboard` becomes the new grid view.

## 4. Risks & NFRs
*   **Migration:** Users accustomed to folders might feel lost. **Mitigation:** "My Organizations" panel remains available.
*   **Performance:** Fetching metadata for *all* themes might be slow if a user has 100+ themes. **Mitigation:** Pagination + Lightweight initial query.
*   **Breaking Changes:** The URL structure `/organizations/:id` might become secondary or hidden behind the "Context Panel".
